## 贡献指南

1. 每个重要更新都应该创建一个单独的更新日志文件，放在 `docs/changelogs/` 目录下
2. 文件命名格式：`YYYY-MM-DD-feature-name.md`
3. 在主更新日志（本文件）中添加对应版本的简要说明
4. 在具体的更新日志文件中提供详细的更改说明
